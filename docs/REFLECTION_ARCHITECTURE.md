# Reflection Architecture

Reflection is the post-execution intelligence layer of JARVIS OS.

## Flow

1. Executive JARVIS receives the request.
2. Reasoning builds the decision path before execution.
3. Workflow or task execution runs.
4. Reflection compares the expected result with the actual result.
5. Learning metadata is prepared for future reasoning.
6. Memory and Knowledge remain the permanent storage systems.

## Design Rules

- Executive JARVIS remains the decision authority
- Provider Router remains the only provider gateway
- Reflection is advisory, not autonomous
- Reflection is deterministic and architecture-only in this version
