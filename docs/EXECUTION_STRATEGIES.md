# Execution Strategies

Supported execution strategies are metadata only:

- `direct`
- `memory_first`
- `knowledge_first`
- `task_first`
- `plugin_first`
- `single_provider`
- `multi_provider`
- `parallel`
- `sequential`
- `fallback`
- `recovery`
- `local_first`
- `cloud_first`
- `hybrid`

These strategies guide future execution planning without changing Provider Router ownership.
